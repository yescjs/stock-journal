# Specification: UI/UX 전면 개편 (토스 스타일)

## 1. 개요 (Overview)
본 트랙은 Stock Journal 애플리케이션의 UI/UX를 '토스(Toss)' 스타일로 전면 개편하는 것을 목표로 합니다. 간결하고 직관적인 모바일 경험을 최우선으로 하며, 부드러운 인터렉션과 현대적인 핀테크 감성을 제공하여 사용성을 극대화합니다.

## 2. 주요 목표 (Key Goals)
- **Toss Style Identity**: 둥근 모서리, 부드러운 그림자, 선명한 블루 포인트 컬러, 넉넉한 여백을 적용합니다.
- **Micro-interactions**: 버튼 클릭, 모달 진입, 페이지 전환 시 자연스러운 애니메이션을 적용합니다.
- **Adaptive Layout**: 데스크탑에서도 모바일 경험을 해치지 않는 '하이브리드 레이아웃'을 구현합니다.
- **Theme System**: 시스템 설정을 따르는 완벽한 라이트/다크 모드를 지원합니다.

## 3. 기능적 요구사항 (Functional Requirements)

### 3.1. 디자인 시스템 재정의 (Design System Refresh)
- **Typography**: 가독성이 높은 산세리프 폰트(Pretendard 또는 시스템 폰트) 적용. 크고 굵은 헤드라인과 명확한 계층 구조 확립.
- **Color Palette**:
  - Primary: Toss Blue 계열의 선명한 파란색.
  - Background: (Light) 웜 그레이/화이트, (Dark) 딥 그레이/블랙.
  - Semantic: 상승(Red), 하락(Blue) 색상은 주식 시장 관례를 따르되, 파스텔 톤 배경과 조합하여 세련되게 표현.
- **Shape & Elevation**: 모든 카드와 버튼에 `rounded-2xl` 이상의 둥근 모서리 적용. 부드럽고 넓게 퍼지는 그림자(Elevation) 사용.

### 3.2. 핵심 컴포넌트 스타일링 (Component Styling)
- **Core Controls**:
  - 버튼: 꽉 찬 배경(Solid), 연한 배경(Light), 텍스트(Text) 스타일 재정의. 터치 영역 확보.
  - 입력창: 테두리가 없거나 매우 얇고, 포커스 시 부드러운 하이라이트. 배경색을 활용한 입력 영역 구분.
- **Data Display**:
  - 카드: 콘텐츠를 담는 기본 단위. 패딩을 넉넉하게 주고 그림자로 계층 구분.
  - 리스트: 구분선보다는 여백으로 항목 분리.
- **Navigation**:
  - 하단 탭 바: 아이콘 중심의 심플한 디자인, 선택 시 미세한 햅틱/애니메이션 피드백.
  - 헤더: 스크롤에 따라 투명도가 변하거나 축소되는 인터렉션 적용.

### 3.3. 레이아웃 및 인터렉션 (Layout & Interactions)
- **Hybrid Layout**:
  - **Mobile**: 100% 너비 활용.
  - **Desktop**: 메인 콘텐츠는 모바일 너비(약 480px~600px)로 중앙 정렬. 좌우 여백에는 글로벌 네비게이션, 프로필 요약, 시장 지수 등 보조 정보 배치.
- **Transitions**: `Framer Motion`을 활용하여 페이지 이동 시 부드러운 슬라이드/페이드 효과 적용. 바텀 시트는 아래에서 위로 부드럽게 올라오는 스프링 애니메이션 적용.

### 3.4. 테마 시스템 (Theming)
- `next-themes` 등을 활용하여 시스템 설정(OS)에 따라 라이트/다크 모드 자동 전환.
- Tailwind CSS의 `dark:` 클래스를 전면적으로 활용하여 모든 컴포넌트가 두 모드에서 자연스럽게 보이도록 구현.

## 4. 비기능적 요구사항 (Non-Functional Requirements)
- **성능**: 애니메이션 추가로 인한 렌더링 성능 저하가 없어야 함 (Lighthouse 점수 유지).
- **접근성**: 색상 대비 명도 준수 및 스크린 리더 호환성 유지.

## 5. 승인 기준 (Acceptance Criteria)
- [ ] 모든 페이지(로그인, 대시보드, 매매일지 등)가 토스 스타일 가이드라인(폰트, 컬러, 여백, 쉐이프)을 준수해야 한다.
- [ ] 버튼 클릭, 모달 오픈 시 지정된 애니메이션이 끊김 없이 동작해야 한다.
- [ ] 데스크탑 브라우저에서 실행 시, 중앙 컨테이너 구조의 하이브리드 레이아웃이 정상적으로 표시되어야 한다.
- [ ] 시스템 테마 설정을 변경했을 때, 앱의 색상이 즉시 올바르게 반영되어야 한다.
