# 명세서 - 외부 레퍼지토리 및 임시 폴더 정리

**1. 개요 (Overview)**
- **트랙명**: 외부 레퍼지토리 및 임시 폴더 정리 (`cleanup_external_repos`)
- **목적**: 현재 프로젝트 루트에 존재하는 불필요한 외부 레퍼지토리 폴더들을 정리하여 프로젝트 구조를 최적화하고 용량을 확보합니다.
- **대상 폴더**: `fast-xml-parser-temp`, `n8n-temp`, `rss-parser-temp`, `RSSHub-temp`

**2. 기능적 요구사항 (Functional Requirements)**
- **폴더 분석**: 삭제 전 각 대상 폴더의 내부 구조와 파일을 점검하여 `stock-journal` 프로젝트와 관련된 중요 설정이나 코드가 포함되어 있는지 확인합니다.
- **안전한 삭제**: 분석 결과 특이사항이 없을 경우, 대상 폴더 4개를 로컬 디스크에서 완전히 삭제합니다.
- **결과 보고**: 삭제 완료 후 제거된 폴더 목록과 확보된 용량(또는 결과)을 사용자에게 알립니다.

**3. 비기능적 요구사항 (Non-Functional Requirements)**
- **안정성**: 실수로 프로젝트 본연의 파일(app, components, lib 등)이 삭제되지 않도록 주의합니다.
- **투명성**: 분석 과정에서 의심스러운 파일이 발견되면 사용자에게 즉시 보고하고 판단을 구합니다.

**4. 수락 기준 (Acceptance Criteria)**
- [ ] 대상 폴더 4개가 프로젝트 루트 디렉토리에서 완전히 제거됨.
- [ ] 삭제 전 모든 폴더에 대한 내용 분석이 선행됨.
- [ ] 삭제 후 `stock-journal` 프로젝트의 빌드 및 실행에 영향이 없어야 함.

**5. 범위 제외 (Out of Scope)**
- `.next`, `node_modules` 등 프로젝트 필수 폴더 정리.
- Git 설정(` .git` 폴더) 수정.
