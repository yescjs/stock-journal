# 매매일지 서비스 수요 검증 실험 설계

## 배경

매매일지.com은 기능적으로 풍부하지만(매매 기록, AI 분석, 뉴스, 트레이딩 도구), 사용자가 거의 없는 상태(0~5명). 제품을 더 만들기 전에 **"한국 개인투자자 중 매매일지를 웹 서비스로 쓰고 싶은 사람이 존재하는가?"**를 먼저 검증한다.

## 핵심 가설

> 한국 개인투자자 중 엑셀/Notion 대신 전용 웹 매매일지를 사용할 의향이 있는 사람이 존재한다.

## 시장 조사 요약

- **해외**: Tradervue(20만+), TraderSync(10만+), TradesViz(10만+) 등 검증된 니치 SaaS 카테고리
- **한국**: 전문 서비스 사실상 부재. 소규모 앱만 존재(다운로드 수백 건 수준)
- **현실**: 1,410만 개인투자자 중 매매일지를 적극 작성하는 비율은 극소수. 무료 대안(엑셀, Notion) 풍부
- **기회**: 한국 증권사 연동 + AI 분석을 갖춘 전문 서비스가 없음

## 검증 전략: 커뮤니티 반응 + GA4 트래킹 병행

빠른 정성적 피드백(커뮤니티 댓글)과 정량적 데이터(GA4)를 동시에 수집하여 2주 안에 판단한다.

## 파트 1: GA4 설치 + 전환 이벤트

### 설치 방식

- `@next/third-parties`의 `GoogleAnalytics` 컴포넌트 사용 (Next.js 공식 지원)
- `app/[locale]/layout.tsx`에 추가 (기존 Vercel Analytics 옆)
- 환경변수: `NEXT_PUBLIC_GA_MEASUREMENT_ID`

### 전환 이벤트

| 이벤트 | 트리거 시점 | 목적 |
|--------|------------|------|
| `page_view` | 자동 (GA4 기본) | 어떤 페이지에 유입되는지 |
| `sign_up` | 회원가입 완료 시 | 가입 전환율 |
| `guest_start` | 게스트 모드 시작 시 | 비가입 전환율 |
| `first_trade` | 첫 거래 기록 시 | 핵심 활성화 지표 |
| `landing_cta_click` | 랜딩 페이지 CTA 클릭 시 | 랜딩 -> 서비스 전환 |

### 구현 범위

- GA4 스크립트 삽입: `app/[locale]/layout.tsx` (1개 파일)
- `gtag` 유틸 함수: 새 파일 `lib/gtag.ts`
- 기존 hook에 이벤트 전송 추가: `useSupabaseAuth`, `useTrades` 등 (각 1~2줄)

## 파트 2: 랜딩 페이지 가치 제안 명확화

### 현재 상태

- 태그라인: "매매 일지 . AI 투자 코칭 . 차트 분석 . 클라우드 동기화" (기능 나열)
- 헤드라인: "투자 기록이 실력이 됩니다." (추상적)
- CTA: "무료로 시작하기"

### 개선 방향

| 항목 | 현재 | 개선 |
|------|------|------|
| 태그라인 | 기능 나열 (4개) | 타겟 명시: "개인 투자자를 위한 무료 매매일지" |
| 헤드라인 | 추상적 ("실력이 됩니다") | 구체적 pain point 또는 유지 |
| CTA | "무료로 시작하기" | "가입 없이 바로 시작" (게스트 모드 강조) |

### 핵심 원칙

- 방문자가 3초 안에 "이게 나한테 필요한 건지" 판단할 수 있어야 함
- 게스트 모드(가입 불필요)를 전면에 내세워 진입 장벽 최소화
- 기존 컴포넌트 구조 유지, 번역 파일 텍스트만 수정

### 변경 파일

- `messages/ko.json` hero 섹션
- `messages/en.json` hero 섹션
- `LandingPage.tsx` CTA 버튼 순서/텍스트 조정

## 파트 3: 커뮤니티 수요 검증 글

### 대상 커뮤니티

| 커뮤니티 | 특성 | 글 형태 |
|----------|------|---------|
| 에펨코리아 주식 게시판 | 개인투자자 활발, 홍보 관용적 | 후기/공유형 |
| 클리앙 주식 게시판 | IT 관심층, 품질 중시 | 개발 후기형 |
| 디시 주식갤러리 | 트래픽 높음, 반응 빠름 | 짧고 직관적 |
| Reddit r/korea or r/daytrading | 영어권 + 해외주식 투자자 | 사이드 프로젝트 공유형 |

### 글 톤: "직접 만들어 봤습니다"

홍보글이 아닌 개발자의 사이드 프로젝트 공유 형태.

글 구조:
1. 동기: "엑셀로 매매일지 쓰다가 귀찮아서 직접 만들었습니다"
2. 기능 소개: 스크린샷 2~3장 (캘린더 뷰, AI 분석, 트레이딩 도구)
3. 차별점: "가입 없이 바로 써볼 수 있습니다 (게스트 모드)"
4. 링크: 매매일지.com (UTM 파라미터 포함)
5. 피드백 요청: "이런 서비스 수요가 있을까요?"

## 성공/실패 기준

### 커뮤니티 반응

| 지표 | 실패 | 보통 | 성공 |
|------|------|------|------|
| 글 조회수 | <100 | 100~500 | 500+ |
| 링크 클릭 (GA4) | <5 | 5~20 | 20+ |
| 가입/게스트 시작 | 0 | 1~5 | 5+ |
| 댓글 톤 | 무관심/부정 | 혼재 | 긍정+기능 요청 |
| 1주 후 재방문 | 0 | 1~2 | 3+ |

### 2주 후 판단

| 결과 | 판단 | 다음 행동 |
|------|------|-----------|
| 성공 (가입 5+, 재방문 3+, 긍정 댓글) | 수요 존재 확인 | SEO 강화 + 바이럴 기능 개발로 전환 |
| 보통 (가입 1~5, 관심은 있지만 미미) | 수요 약함, 방향 조정 필요 | 피드백 기반 핵심 기능 피벗 검토 (예: 도구 중심 전환) |
| 실패 (클릭 <5, 가입 0, 무관심) | 수요 부재 | 매매일지 방향 중단, 다른 가치 제안 탐색 |
